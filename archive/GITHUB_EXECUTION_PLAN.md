# GITHUB_EXECUTION_PLAN.md · GitHub 落地执行方案

> 版本：Core V2.0 | 更新时间：2026-04-27 18:34 CST | 维护：太阳

## A. 上传前检查清单
- 是否明确本次属于：升级上传 / 新增上传 / 有回退风险
- 是否只包含允许公开的文档/配置说明/规则文件
- 是否包含 `.env`、API Key、token、账号密码、私钥、客户资料：如有，禁止上传
- 是否保留原有内容基础，不做破坏式重建
- GitHub 页面最终可见结果是否与本次目标一致

## B. 公开仓库禁传项
- `.env`
- API Key / token / cookie / SSH key
- 客户资料 / 对话记录 / 订单信息 / KPI 数据
- 本地运行缓存 / 日志 / 数据库文件
- 未清洗原始资料

## C. 分工
- 太阳：定是否上传、定上传类型、定保留/删除、做最终验收
- 光头强：执行检查、上传、同步、公开/私有设置、技术排障

## D. 忙不过来时交接
- 太阳给出一句明确任务：上传什么、传到哪、属于哪种上传
- 光头强按检查清单执行
- 执行后只回：已做什么、已完成什么、下一步核查什么

## E. 最小可执行流程
1. 本地整理完成
2. 判断：升级上传 / 新增上传 / 有回退风险
3. 做机密检查
4. 确认保留原有内容
5. 上传到 GitHub
6. 刷新 GitHub 页面验收
