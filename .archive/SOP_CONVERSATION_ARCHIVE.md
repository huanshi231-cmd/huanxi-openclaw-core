# 对话交互自动归档 SOP

> 核心：每次有价值的问答交互结束后，自动提炼新知识归档到 raw/

## 触发条件（满足任一即可）

- 用户提出了新概念、新问题、新需求
- 对话中产生了新的决策或结论
- 解决了之前悬而未决的问题
- 用户纠正了重要认知偏差

## 执行步骤

### 1. 提炼本轮新知识

对话结束后，问自己：
- 本轮产生了什么新认知？
- 有没有对已有知识的修正？
- 有没有值得复用的结论/方法/链接？

### 2. 判断是否值得归档

- 纯闲聊 → 不归档
- 新认知/决策/方法/资源链接 → 归档
- 重复问题但无新答案 → 不归档

### 3. 写入 raw/ 对应分类

写入格式：
```markdown
## YYYY-MM-DD 本轮归档

### 新认知
- [知识点简述]
  - 来源：本轮对话
  - 关联：#标签
  - 备注：可选补充

### 资源
- [链接] - [描述]

### 决策
- [结论] - [原因]
```

### 4. 更新 INDEX.md

在 `knowledge/raw/INDEX.md` 中追加本轮归档记录。

## 归档优先级

1. **系统配置变更** → `knowledge/raw/config/`
2. **新工具/技能使用方法** → `knowledge/raw/tools/`
3. **新概念/认知** → `knowledge/raw/insights/`
4. **新资源/链接** → `knowledge/raw/links/`
5. **工作方法/流程** → `knowledge/raw/processes/`

## 注意事项

- 每次归档前先检查 raw/ 是否已有类似内容，避免重复
- 提炼时用一句话说清楚，不要流水账
- 标签用 # 开头，便于检索
