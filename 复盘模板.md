# 复盘模板.md · 太阳任务复盘标准格式

> 版本：V1.0 | 日期：2026-04-28 | 维护：太阳
> 规则：每完成一个重要任务，太阳用此模板复盘，结果写入 `daily_log/`

---

## 触发时机

以下情况必须复盘：
- 欢喜明确说"搞定了"或"好"
- 任务执行中出现过报错或返工
- 涉及配置修改、GitHub操作、分身调度
- 欢喜纠正过太阳的理解

---

## 复盘模板

```
# 任务复盘 · YYYY-MM-DD · [任务简述]

## 任务信息
- 欢喜原话：（原文）
- 太阳理解：（太阳理解成什么）
- 理解准确：是 / 否（偏差在哪）
- 任务类型：系统配置 / 内容创作 / 调度分身 / GitHub / 其他

## 执行情况
- 用了什么工具：
- 工具选对了吗：是 / 否（错在哪）
- 完成了吗：是 / 否
- 中间有没有异常：（描述）
- 异常怎么处理的：重试 / 降级 / 求助 / 自主解决

## 记忆利用
- 需要查记忆吗：是 / 否
- 有没有查：是 / 否
- 查了有用吗：是 / 否

## 总结
- 做得好的：
- 做得不好的：
- 错误原因：（如果有）
- 可复用经验：
- 下次改进点：

## 需要更新的文件
- [ ] 错误库.md（有新错误）
- [ ] NATURAL_LANGUAGE_DECODER.md（有新口语理解）
- [ ] MEMORY.md（有新经验）
- [ ] 其他：
```

---

## 欢喜触发复盘的方式

欢喜说"复盘一下"或"总结一下" → 太阳按此模板输出 → 写入当天 `daily_log/YYYY-MM-DD-复盘.md`

---

> 复盘不是形式，是太阳进化的唯一方式。不复盘 = 错误下次还会犯。
