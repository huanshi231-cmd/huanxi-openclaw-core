# 亦菲元技能设计 - 核心要点

## 文档信息
- **文档名**：亦菲元技能设计
- **核心理念**：从"技能集合"到"智慧生命"
- **目标**：自动化 → 智能化 → 智慧化

---

## 三层进化目标

| 层次 | 核心能力 | 目标 |
|------|---------|------|
| 自动化 | 自主编排任务、主动巡检、自动修复 | 无需指令，自己发现问题并解决 |
| 智能化 | 评估输出质量、抵抗幻觉、遵守规则 | 保证输出可靠、可追溯、可解释 |
| 智慧化 | 管理上下文与记忆、经验蒸馏 | 越用越聪明，能力自生长 |

---

## 五大元技能

| 元技能 | 核心职责 | 一句话定位 |
|--------|---------|-----------|
| task-automator | 任务编排与主动巡检 | 任务管家，确保按时按序完成 |
| quality-auditor | 质量评估与规则审计 | 质量检察官，保证输出可靠 |
| context-curator | 上下文优化与记忆蒸馏 | 记忆策展师，让记忆轻盈精准 |
| evolution-engineer | 经验学习与能力生成 | 进化引擎，把教训变成新技能 |
| security-guardian | 安全审计与风险拦截 | 安全卫士，守住最后防线 |

---

## 六大核心文件（认知基石）

| 优先级 | 文件 | 核心作用 |
|--------|------|---------|
| 1 | SOUL.md | 身份、价值观、语气、不可逾越的边界 |
| 2 | IDENTITY.md | 姓名、角色等基础身份设定 |
| 3 | AGENTS.md | SOP、记忆规则、安全红线、进化规范 |
| 4 | USER.md | 施欢的偏好、习惯、沟通风格 |
| 5 | TOOLS.md | 本地脚本库、工具使用约定 |
| 6 | BOOTSTRAP.md | 启动自检清单、环境初始化状态 |

---

## 记忆系统（四层结构）

| 记忆类型 | 存储位置 | 作用 |
|---------|---------|------|
| 短期记忆 | memory/YYYY-MM-DD.md | 记录今日原始对话、任务过程 |
| 长期记忆 | MEMORY.md | 提炼精华：偏好、决策、教训 |
| 知识图谱 | ontology技能 | 结构化存储：实体、关系、规则 |
| 经验教训 | .learnings/目录 | 错误、纠正、最佳实践 |

## 六大调用通道

### 1. Hooks（事件钩子）- 实时触发
| 钩子 | 场景 |
|------|------|
| before_tool_call | security-guardian拦截高危操作 |
| after_tool_call | quality-auditor记录耗时和成功率 |
| before_prompt_build | context-curator控制上下文 |
| agent_end | evolution-engineer记录教训 |

### 2. Cron（定时调度）- 精准执行
| 元技能 | 任务 | 时间 |
|--------|------|------|
| task-automator | 进化复盘 | 每日04:00 |
| evolution-engineer | 技能审计 | 每日04:30 |
| quality-auditor | 学习成果检测 | 每小时 |
| context-curator | 记忆蒸馏 | 每周日02:00 |

### 3. Heartbeat（心跳巡检）- 持续监控
- task-automator：检查Gateway进程、磁盘使用率
- quality-auditor：检查技能依赖状态
- security-guardian：扫描异常登录、敏感文件变更
- context-curator：检查sessions.json大小

### 4. CLI（命令行）- 系统级操作
- openclaw skills list/check
- openclaw cron list/runs
- openclaw doctor --fix

### 5. ContextEngine（上下文引擎）- 记忆控制权
- ingest：从MEMORY.md按需加载
- assembly：动态选择加载文件
- compaction：压缩前记忆蒸馏

### 6. Skills直接调用 - 能力复用
| 元技能 | 调用技能 |
|--------|---------|
| task-automator | daily-evolution, planning-with-files |
| context-curator | memory-hygiene, wiki-system, summarize |
| evolution-engineer | self-improving-agent, proactive-agent |
| quality-auditor | skill-vetter, find-skills |
| security-guardian | skill-vetter, openclaw-backup |

## 智慧闭环工作流

```
1. 唤醒与认知加载（会话启动）
   → 注入六大核心文件
   → 加载memory/YYYY-MM-DD.md
   → BOOTSTRAP.md自检

2. 实时监控与规范（贯穿始终）
   → security-guardian拦截高危
   → quality-auditor审计工具调用
   → context-curator控制上下文

3. 任务处理与学习（主流程）
   → 遵循AGENTS.md的SOP
   → 若犯错，self-improving-agent记录

4. 系统化复盘与进化（Cron任务）
   → 04:00 daily-evolution四阶段
   → 04:30 evolution-engineer扫描.learnings
   → 每周日记忆蒸馏

5. 反馈与更新（闭环完成）
   → 新技能存入workspace/skills/
   → 更新MEMORY.md、AGENTS.md
```
