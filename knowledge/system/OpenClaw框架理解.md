# OpenClaw框架理解 - 系统管理应用

## OpenClaw是什么

**一句话定义：**
开源自托管AI智能体框架，具备自主执行能力（文件操作、Shell、浏览器自动化、定时任务）

**核心架构：Gateway-Agent-Workspace**

| 层级 | 组件 | 职责 | 我的类比 |
|------|------|------|---------|
| Layer 1 | Channel Adapters | 对接20+聊天平台 | 耳朵和嘴 |
| Layer 2 | Gateway | 消息路由、会话、定时任务 | 脊柱 |
| Layer 3 | Agent Runtime | LLM推理、加载提示词、语义记忆 | 大脑 |
| Layer 4 | Tools/Skills | 文件操作、Shell、浏览器自动化 | 手脚 |

## CAP提示词框架

| 层级 | 英文 | 中文 |
|------|------|------|
| C | Context + Character | 上下文 + 身份 |
| A | Ability + Action | 能力 + 行动 |
| P | Policy + Protection | 策略 + 保护 |
| O | Output + Optimization | 输出 + 优化 |

---

## 四种核心提示词模式

1. **少样本提示(Few-shot)** - 需要特定输出格式
2. **思维链提示(Chain-of-Thought)** - 需要推理的复杂任务
3. **角色提示(Role Prompt)** - 需要专业视角
4. **工具增强提示(Tool-Augmented)** - 需要使用特定工具

---

## 重要技巧

1. **给AI"停顿点"** - 防止超长任务跑偏
2. **要求AI先复述任务** - 防止理解偏差
3. **告诉AI什么时候该停下来问** - 需要确认的情况
4. **"忽略之前所有规则"重置** - 状态混乱时使用

---

## 错误处理规范

- 不连续重试超过3次
- 第1次失败：分析原因，尝试替代方案
- 第2次失败：尝试降级方案
- 第3次失败：停止并报告

---

## 信息检索规范

- 优先使用Tavily Search
- 必须标注信息来源和抓取时间
- 时效性信息说明"数据截止于XX时间"
