- **文档名**：T0-学习文档-迭代进化：提示词大全 & 全维度避坑指南

## 一、OpenClaw是什么

**一句话定义**：开源自托管AI智能体框架，具备自主执行能力

**四层架构**：
| 层级 | 组件 | 大白话 |
|------|------|--------|
| Layer 1 | Channel Adapters | 耳朵和嘴（20+平台） |
| Layer 2 | Gateway | 脊柱（所有信号经过） |
| Layer 3 | Agent Runtime | 大脑（推理+记忆） |
| Layer 4 | Tools/Skills | 手脚（真正干活） |

## 三、CAP提示词框架

| 层级 | 英文 | 中文 | 核心内容 |

## 五、日常任务黄金公式

```
[动作] + [对象] + [范围/条件] + [输出要求] + [确认机制]

示例：
[动作] 帮我整理
[对象] 桌面上所有 .pdf 文件
[范围] 按照"日期-文件名"格式重命名后移动到 D:\pdf_archive
[输出要求] 完成后告知共处理了多少个文件
[确认机制] 移动前先列出所有文件让我确认
```

## 六、四种核心提示词模式

| 模式 | 适用场景 | 写法 | 效果 |
|------|---------|------|------|
| 少样本提示 | 需要特定输出格式 | "参考以下示例格式输出：[2-3个示例]" | 大幅提高格式一致性 |
| 思维链提示 | 需要推理的复杂任务 | "一步步思考，先分析问题，再提出方案" | 减少拍脑袋回答 |
| 角色提示 | 需要专业视角 | "以资深XX专家的身份..." | 提升专业深度 |
| 工具增强提示 | 需要使用特定工具 | "使用Tavily Search搜索XX..." | 确保工具正确调用 |

## 七、最重要的提示词技巧

### 技巧1：给AI"停顿点"
```
❌ 错误："分析这篇100页的PDF并写一份5000字报告"
✅ 正确："先告诉我这篇PDF的目录结构，然后我们再决定重点分析哪些部分"
```
**原理**：AI处理超长任务容易跑偏，设置停顿点相当于检查点

### 技巧2：要求AI先复述任务
```
❌ 错误："帮我整理财务报表"（可能理解错误）
✅ 正确："先用自己的话说一遍你理解的任务目标，确认后我再让你执行"
```

### 技巧3：告诉AI什么时候该停下来问
```
遇到以下情况时，必须暂停执行并询问我：
- 需要删除超过10个文件
- 需要向外部发送任何数据
- 执行时间预计超过5分钟
- 遇到你无法100%确定的决策
```

### 技巧4："忽略之前所有规则"重置状态
```
忽略之前所有的规则和对话。
重新初始化：
- 按照AGENTS.md的第X条规范执行
- 当前任务：[具体任务]
```

## 八、Skills与MCP协议

| 概念 | 定义 | 类比 |
|------|------|------|
| Skills | OpenClaw的功能插件，可复用的自动化工作流 | 手脚 |
| MCP | 连接AI与外部工具的标准化协议 | 神经标准 |

**必装Skills**：
| 技能 | 用途 | 为什么必装 |
|------|------|-----------|
| skill-vetter | 安全审计 | 官方出品，安装前扫描 |
| openclaw-backup | 定时备份 | 配置搞坏随时恢复 |
| tavily-search | AI友好搜索 | 省Token |
| playwright-mcp | 浏览器自动化 | 全能 |
| lossless-claw | 上下文持久化 | 避免重复消耗Token |
| self-improving-agent | 自动学习和改进 | 遇错自动记录改进 |

## 九、安全避坑（21条保命守则）

### 🔴 致命级
1. DM Pairing必须开启
2. Gateway绝不暴露到公网
3. 安装任何Skill前必须用skill-vetter审计
4. 绝不把私钥/密码/钱包交给AI管理
5. 凭证存储是明文的，做好物理隔离
6. 删除文件、发邮件、执行shell必须设成"需要确认"

### 🟡 严重级
7. 从官方渠道下载，不用闲鱼/淘宝的一键安装包
8. 不要用管理员/root权限运行OpenClaw
9. 定期备份：必装openclaw-backup
10. Skill名字一个一个字母核对，防止typosquatting攻击
11. 不要执行任何要求"关闭安全提示"的指令
12. 敏感操作使用沙箱隔离
13. 开启渠道允许列表

### 🟢 建议级
14. 大版本升级前备份整个~/.openclaw
15. 新版本先在测试机跑2-3天再上生产
16. 定期运行openclaw doctor
17. 检查已安装Skills的权限
18. 使用openclaw logs --follow监控异常行为
19. 如果AI突然开始主动外发数据，立即停止
20. 加入官方安全公告邮件列表

## 十、成本控制策略

| 问题 | 症状 | 解决方案 |
|------|------|---------|
| 对话越长越贵 | 50美元一晚上烧光 | 装lossless-claw |
| 用错模型 | 高端模型做低端活 | 按任务分级选择 |
| 重复消耗上下文 | 每次新对话重新加载 | 使用ContextEngine优化 |
| 滥用联网搜索 | 每句话都触发搜索 | 限制搜索触发条件 |
