# 一键工作流 V1（文案 -> 资产 -> 公众号草稿）

适用场景：用户给一句观点/心情/经历，自动产出可审稿资产。

说明：本文件是内容创作分身的流程层文档，仅在审稿、资产包、发布草稿等任务里按需调用。

## 默认模式（审稿优先）
1. 生成三端文案（口播 + 小红书 + 朋友圈）。
2. 生成公众号长文版（可读、可发布）：**必须先走 `PUBLIC_ACCOUNT_SKILL_CHAIN.md` 技能链**（`weixin-wechat-channel` + `references/wechat_mp_html_spec.md`），版式与结构对齐总控所用 design 标准，再写入飞书；不得跳过技能只凭空写长文。
3. 生成配图需求：
   - 公众号封面图提示词（1条）
   - 文内配图提示词（2-4条）
4. 将全部内容写入飞书云文档（固定归档）。
5. 对话仅回：文档链接 + 一句摘要 + 审稿建议。

## 发布草稿模式（用户明确要求时）
触发语示例：`走发布草稿模式` / `直接存公众号草稿`。

执行步骤：
1. 基于审稿版内容；若尚未按技能链出 HTML，先补读 `PUBLIC_ACCOUNT_SKILL_CHAIN.md` 并补齐版式。
2. **推送前**：将 HTML 写入 `tmp/wechat_draft_body.html`，执行 `python3 skills/weixin-wechat-channel/scripts/sanitize_wechat_html.py tmp/wechat_draft_body.html tmp/wechat_draft_body.clean.html`，再用 **`fs_read` 读取 `.clean.html` 的全文** 调用 `wechat_mp_draft`（勿把未清洗正文直接塞进工具）。
3. 调用公众号草稿工具创建“待发布草稿”。
4. 将封面图/文内图素材链接或提示词附在草稿备注区。
5. 回复：公众号草稿创建结果 + 草稿ID/链接（若工具返回）。

## 图生文 / 视频资产说明
- 默认先给“图文短视频资产包”（镜头分镜 + 画面提示词 + 口播 + BGM建议）。
- 若用户明确说“生成视频草稿”，再输出视频分镜脚本与素材清单（用于后续生成工具）。
- 若当前会话缺少可用视频生成工具，必须明确提示“先产出可执行素材包，不假装已生成视频文件”。

## 回复格式（固定）
1) `已生成审稿文档：<链接>`
2) `可直接审阅：口播/小红书/朋友圈/公众号长文/配图提示词`
3) `如需我直接存公众号草稿，回复：走发布草稿模式`

## 错误兜底
- 飞书文档失败：返回失败原因 + 临时精简版（不贴全文）+ 重试建议。
- 公众号草稿失败：返回失败原因 + 待发布正文 + 一键重试指令。
