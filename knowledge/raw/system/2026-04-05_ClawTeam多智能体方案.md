# ClawTeam — 多智能体终极解决方案

## 核心价值

将 OpenClaw 从单智能体转为多智能体"蜂群"平台。

## 解决的核心痛点

| OpenClaw 原生问题 | ClawTeam 解决方案 |
|---|---|
| 请求响应模式，无持久状态 | 持久化任务状态 |
| Subagent 共享工作区，无代码隔离 | 团队隔离 |
| 子智能体之间无法直接通信 | 点对点消息广播 |
| 超时后强制终止 | 无超时限制，可运行数小时 |

## 核心功能

1. **自然语言任务描述**：一句话自动启动研究团队
2. **智能拆分与组建**：根据任务复杂度自动组建多 Agent
3. **持久化看板**：任务看板 + 依赖自动解锁 + 消息广播
4. **深度研究能力**：无超时限制

## 系统架构

1. 用户自然语言输入目标
2. 任务到达 OpenClaw 领导者智能体
3. 领导者根据复杂度派生多种智能体
4. 各智能体共享：团队配置、任务看板、计划审批、团队隔离

## 安装方式

- 自动安装：对 OpenClaw 说「阅读这个项目的安装说明，并为我安装这个项目」+ 仓库链接
- 手动安装：执行 README 命令
- 重启网关：`opencloud gateway restart`

## 评估

**用途**：解决多分身协作效率问题（当前 system→neirong 派单靠文件+等待，实时性差）

**安装必要性**：高——可补足当前 P2P 通信机制的实时性短板

**风险**：待评估（需读原项目代码确认安全性）
