# LLM 自动编译流程

## 触发时机

每次 neirong 完成创作任务后，触发「归档 → 编译」闭环。

## 流程

### 1. 收集原始素材
读取本次使用过的所有素材，来源包括：
- `knowledge/raw/temp/` 新增文件
- 飞书文档引用的链接
- 创作过程中产生的中间笔记

### 2. LLM 自动处理

对每个素材执行：

```
输入：原始文件路径 / 链接内容
处理：
1. 生成摘要（50字内）
2. 提取标签（来源、主题、类型）
3. 确定分类目录
4. 记录「用于：本次创作成品名」

输出：写入 `knowledge/raw/INDEX.md`
```

### 3. 建立反向链接

在本次创作成品文档中，追加「本篇使用的素材清单」，指向 `knowledge/raw/` 中的对应文件。

### 4. 多格式输出（按需）

- 创作结论 → 自动生成 Marp 格式摘要幻灯片
- 关键数据 → 生成 Matplotlib 可视化（写入 `knowledge/raw/`）
- 长文 → 自动生成简短摘要存入 INDEX

## 示例

归档完成后，在 INDEX.md 中追加：

```markdown
## 2026-04-04

- 水晶疗愈能量图片 - 来源：Pexels - 用于：清明家族树疗愈视频 v2
  - 标签：#水晶 #能量符号 #免费图
  - 反向链接：← 清明家族树疗愈视频 v2

- 女性成长爆款公式库 - 来源：飞书云文档 - 用于：本周公众号选题
  - 标签：#女性成长 #爆款公式 #选题库
  - 反向链接：← 2026-04-04 公众号选题清单
```

## 健康检查 SOP

每月执行一次，放在 cron 任务里：

1. 扫描 INDEX.md，检测同一主题有矛盾描述的文件
2. 检测 SOP 文件是否与当前工具链一致（过时则标记）
3. 联网搜索补充该领域最新资讯
4. 推荐新的概念关联
5. 输出检查报告到 `knowledge/raw/HEALTH_CHECK_YYYY-MM.md`
