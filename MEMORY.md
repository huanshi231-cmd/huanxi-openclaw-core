## 📌 组织架构原则（2026-04-29 欢喜明确）

### 信息传递规则
- **T+1 制度**：任务从下达到闭环，最多经过1层传递
- **信息流**：欢喜 → 主控（太阳） → 部门TL → 执行员工
- **欢喜只需对话部门TL**，不直接对话执行层
- **TL需要下属时**：TL @下属来说话，欢喜不直接@下属

### 部门TL对照表

| 部门 | TL（负责人） | 职责 |
|------|-------------|------|
| 内容部 | 蕊蕊 | 内容创作、文案、脚本 |
| 运营部 | 跳跳 | 新媒体运营、发布、复盘 |
| 视觉部 | 朵朵 | 封面设计、配图 |
| 技术部 | 光头强 | 系统、工具、自动化 |
| 知识部 | 梦梦 | 记忆库、知识沉淀 |
| 灵感部 | 灵夕 | 选题、灵感、热点 |
| **主控** | **太阳** | COO，统一协调、汇报 |

### 欢喜对话规则
- 欢喜 @谁 → 谁回复
- 需要跨部门 → 太阳协调
- TL需要下属 → TL自己@下属
- 只说给欢喜听 → 不用@，直接说

## 📌 跨部门沟通与任务流转规范（2026-04-29 正式确立）

### 核心原则
1. **T+1 负责人制度**：所有核心讨论、决策同步仅面向各部门第一负责人
2. **禁止越级**：主控（太阳）不直接 @部门二级员工
3. **一级全权路由**：所有下沉到执行的任务，由部门负责人在本部门内拆解、分配

### 流转规范
```
欢喜（CEO）
    ↓ 仅对T+1负责人
太阳（主控/COO）← 唯一对接欢喜的接口
    ↓ 仅对T+1负责人
部门TL（一级负责人）← 部门唯一对外接口
    ↓ 全权路由
部门二级员工（执行层）
```

### 核心目的
- 保证人类讨论聚焦、信息纯净
- 避免无关信息干扰和记忆混乱
- 部门负责人完整掌控本团队执行节奏与信息链路
- 杜绝信息断层和责任不清
