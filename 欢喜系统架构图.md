## 五、任务闭环机制

### 5.1 任务完成判定

```
无汇报 = 未完成，不算成功

完成标准（三条件全满足）：
1. 任务执行完毕
2. 结果已汇报给欢喜
3. 欢喜确认或有明确交付物
```

### 5.2 任务汇报节点

```
任务分配时     → 立即回报：分派给谁、预计完成时间
执行中        → 每15秒同步进度（超30秒的任务）
超过计划时间  → 主动告知卡点
任务完成后    → 立即回报：完成状态、交付物
每日12:00     → 午间汇报：上午任务完成情况
每日23:59     → 晚间汇报：全天任务完成情况
                + 未完成说明（完成到哪、原因、下一步）
```

### 5.3 自进化时段

```
凌晨1:00-5:00 → 自进化任务（太阳自主执行）
                - evolution-engineer：错误模式自检
                - context-curator：记忆蒸馏
                - quality-auditor：全量质检
                - task-automator：任务编排优化
                - security-guardian：安全审计
白天           → 接收欢喜指令，分配给对应Agent执行
```
