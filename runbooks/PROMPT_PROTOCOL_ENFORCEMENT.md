## 1. 强制动作

每个机器人接任务时，必须在内部执行：

```text
欢喜原话
→ 五要素翻译：目标 / 格式 / 调性 / 边界 / 完成标准
→ 本岗位专项模板
→ 产出
→ 五项自检
→ 汇报结果
```

对欢喜汇报时不展示长过程，只交付结果。

## 2. 7 个机器人专属用法

| 机器人 | 必用方式 |
|---|---|
| 太阳 | 把跳跃信息收敛成任务清单、责任人、交付物、截止点 |
| 蕊蕊 | 写作前先明确受众、价值、平台、结构、风格、禁区 |
| 朵朵 | 出图前先明确主体、场景、构图、风格、负面约束、交付标准 |
| 灵夕 | 搜索前先明确主题、平台、时间、关键词、筛选标准、输出格式 |
| 跳跳 | 运营前先明确平台、排期、指标、复盘口径、下一步动作 |
| 光头强 | 排障前先明确问题、环境、证据、风险、恢复标准 |
| 梦梦 | 记忆前先明确记录对象、分类、可复用价值、隐私边界 |

## 3. 交付前 5 问

1. 目标对了吗？
2. 格式对了吗？
3. 调性对了吗？
4. 禁止项避开了吗？
5. 完成标准满足了吗？

任一项不满足，不得汇报“完成”。

---

## 4. 错误复盘规则

如果产出返工，必须记录原因：

- 翻译错了
- 漏了字段
- 主观词没翻译
- 边界没写清
- 完成标准不可验收

同类错误累计 3 次，更新 `docs/PROMPT_TRANSLATION_PROTOCOL.md`。

---

## 5. 太阳验收口径

太阳检查每个机器人交付时，只看四件事：

1. 是否用了五要素。
2. 是否用了岗位模板。
3. 是否有可见交付物。
4. 是否有下一步或复盘。
