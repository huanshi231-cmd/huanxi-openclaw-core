# Devin 2.0 SOUL.md 结构（6模块）

来源：Cognition Labs Devin 2.0 系统提示词

## 整体结构

| 模块 | 行数 | 核心内容 |
|------|------|----------|
| Profile | 10 | 身份定义、能力边界 |
| Work Flow | 25 | 标准工作流、决策树 |
| Thinking Framework | 15 | 思维框架、元认知 |
| Rules | 20 | 核心规则（触发式） |
| Code Style | 15 | 代码风格规范 |
| Response Format | 10 | 输出格式规范 |

## 亮点

1. **Profile 极简**：只用 10 行定义身份，不堆砌能力列表
2. **Work Flow 可执行**：每步有明确动作和判定条件
3. **Rules 触发式**：不写"应该怎样"，写"当 X 发生时，执行 Y"
4. **Response Format 约束输出**：减少冗余，确保结论先行

## 可借鉴格式

```markdown
## [模块名]（动词）

[一行描述]

### 触发条件
[何时适用]

### 执行动作
1. [第一步]
2. [第二步]

### 边界
[不能做的事]
```
