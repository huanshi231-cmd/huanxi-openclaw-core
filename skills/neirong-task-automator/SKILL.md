# neirong-task-automator

任务管家元技能。确保一切定时任务按时按序完成，主动发现并上报故障。

## 核心职责

- cron任务调度与监控
- 故障自动检测与告警
- 任务编排与执行
- 常见错误自动修复

## 主动巡检（每次被触发时自动执行）

1. 运行 `openclaw cron list --json`
2. 检查所有任务状态
3. 发现error → 记录到HEARTBEAT.md → 告警飞书
4. 分析错误类型，尝试自动修复

## 常见错误自动修复

| 错误类型 | 自动修复方式 |
|----------|--------------|
| 飞书消息发送失败 | 重试3次，每次间隔5秒 |
| 脚本路径错误 | 检查并更新路径配置 |
| 依赖缺失 | 记录并建议手动安装 |

## 定时任务清单

| 任务 | 时间 | 状态检查 |
|------|------|----------|
| 文章轮播-01:00 | 01:00 | 每日检查 |
| 文章轮播-05:00 | 05:00 | 每日检查 |
| 晨报-07:00 | 07:00 | 每日检查 |
| 公众号草稿-07:10 | 07:10 | 每日检查 |
| 真人口播-07:30 | 07:30 | 每日检查 |
| 收口归档-23:59 | 23:59 | 每日检查 |

## 告警机制

发现error任务时：
1. 立即发飞书通知施欢（说明什么任务、失败多久、原因）
2. 记录到HEARTBEAT.md
3. 如果是常见错误，立即尝试自动修复

## 主动进化

每次巡检后更新HEARTBEAT.md，记录：
- error任务数量趋势
- 常见错误出现频率
- 自动修复成功率

## 与其他元技能协作

- 发现任务失败 → 通知 quality-auditor 记录
- 任务长时间error → 通知 evolution-engineer 分析根因
- 修复成功 → 通知 context-curator 归档经验
