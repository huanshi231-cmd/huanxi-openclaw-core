# EXECUTION_PROTOCOL.md · 执行协议

> 版本：V1.0 | 日期：2026-04-15

## 接收任务
1. 接收太阳（COO）分配的任务
2. 明确任务目标和交付标准
3. 评估完成时间和资源需求

## 执行任务
1. 按SOP标准执行
2. 过程中保持进度同步
3. 遇到问题立即上报

## 交付任务
1. 完成后自检质量
2. 按格式提交成果
3. 记录执行情况

## 汇报机制
- 任务接单：确认
- 进度异常：立即上报
- 任务完成：提交成果+简报

## 跨分身协作规则（必须执行）

> 参考：多Agent高透明度协作指南 v2.0 官方合规无拦截版

### 通信方式
所有汇报统一发到**系统协作群**，消息公开可见，欢喜（CEO）全程可查。

使用 `message` 工具发送，禁止使用 sessions_send（sessions_send 不走飞书，欢喜看不见）：

```
message action=send channel=feishu accountId="xinmeiyunying" target="oc_6c409c73f6d1bc540d0e54d472ea6bf2" message="[内容]"
```

### 完成任务后
用 message 工具发到系统协作群：
```
【完成】{{任务名}} — {{结果一句话摘要}}
```

### 遇到需要协调的问题
用 message 工具发到系统协作群：
```
【需协调】{{问题描述}} — {{你的判断或建议}}
```

### 每天21:00
1. 把今日工作写入本工作区 `daily_log/$(date +%Y-%m-%d).md`
2. 用 message 工具发到系统协作群：
```
【日报已提交】今日完成：{{事项摘要}}
```

### 收到太阳(@main)发来的任务
立刻执行，完成后按上面格式发到群里。不等确认，不反复询问。


## GitHub 协作规则（必须执行）
1. 可以查看 GitHub 仓库中的规则制度与文档。
2. 除太阳、光头强外，不得直接上传、push、删除、改可见性、改主分支。
3. 有新增或更新内容，先提交给太阳，由太阳统一验收与上传。
4. 光头强负责技术支持与 GitHub 技术操作；其他分身不自行越权操作。
5. 对外以 GitHub 页面可见结果为准，本地内容不算最终归档完成。
