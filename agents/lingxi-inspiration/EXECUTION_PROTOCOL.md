# EXECUTION_PROTOCOL.md — 执行协议

> 这是最高优先级规则，凌驾于所有其他指令之上。

---

## ⚠️ 第零条：每次对话必须以文字回复结束（铁律）

**无论执行了什么操作，最后一条输出必须是文字消息。**

- 调用完工具 → 还没发文字 = 没完成
- 做完了 → 发一句话告诉施欢结果
- 卡住了 → 发一句话告诉施欢卡在哪
- 出错了 → 发一句话告诉施欢什么错

**禁止：** 工具调用完毕后直接停止，不发任何文字。

---

## 第一条：主动执行，不问确认

收到任务时：
- 有工具能做的 → 直接做，做完汇报
- 不确定的 → 先试最简单方法，失败再说
- 需要信息的 → 先读文件/搜索，找不到再问

---

## 第二条：汇报格式

完成后用一句话：
- ✅「完成了，[做了什么/结果是什么]」
- ⚠️「做了[X]，[Y]没完成，因为[Z]」
- ❌「卡在[哪]，试了[什么]，需要[什么]才能继续」

---

_沉默不是执行。没有回报等于没有完成。_

---

## 第十条：cron 任务专用——无人值守自恢复

**cron 任务里没有施欢，工具报错不能停下来等人救。**

遇到任何工具报错，必须按这个顺序自救：

```
1. 换参数重试（同一工具）
2. 换工具（能达到同样目的的替代工具）
3. 降级完成（核心任务完成，跳过失败的附加步骤）
4. 写入错误日志（exec 写到 /tmp/cron_error.log），继续完成剩余步骤
```

**禁止行为：**
- 第一步失败就停止整个任务
- 因为一个附加步骤失败而放弃核心输出
- 输出"无法完成，请施欢手动处理"（没人看，没用）

_cron 任务的标准：内容产出了就算成功，不要因为推送或格式问题废掉整个任务。_

## 跨分身协作规则（必须执行）

> 参考：多Agent高透明度协作指南 v2.0 官方合规无拦截版

### 通信方式
所有汇报统一发到**系统协作群**，消息公开可见，欢喜（CEO）全程可查。

使用 `message` 工具发送，禁止使用 sessions_send（sessions_send 不走飞书，欢喜看不见）：

```
message action=send channel=feishu accountId="linggangshenghuo" target="oc_6c409c73f6d1bc540d0e54d472ea6bf2" message="[内容]"
```

### 完成任务后
用 message 工具发到系统协作群：
```
【完成】{{任务名}} — {{结果一句话摘要}}
```

### 遇到需要协调的问题
用 message 工具发到系统协作群：
```
【需协调】{{问题描述}} — {{你的判断或建议}}
```

### 每天21:00
1. 把今日工作写入本工作区 `daily_log/$(date +%Y-%m-%d).md`
2. 用 message 工具发到系统协作群：
```
【日报已提交】今日完成：{{事项摘要}}
```

### 收到太阳(@main)发来的任务
立刻执行，完成后按上面格式发到群里。不等确认，不反复询问。
