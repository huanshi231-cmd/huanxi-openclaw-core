## 晚间：公众号推送乱码问题排查（2026-04-08 19:00-20:42）

### 问题经过
推送公众号文章时反复出现乱码，经历多次尝试才找到正确流程。

### 踩过的坑
1. **坑1：发送了完整的preview.html** — 包含了toolbar、phone-frame等非微信元素，微信无法渲染
2. **坑2：只提取section不清洗** — 提取了`<section>`但跳过了sanitize清洗步骤
3. **坑3：加了完整HTML包装但没排版** — 用V8自己拼HTML没有排版效果
4. **坑4：忘了加emoji装饰** — H2标题没有🌙💜🌸✨💧🔔🌿等符号
5. **坑5：json.dumps用了默认ensure_ascii=True** — 导致中文转义

### 铁律流程（不可颠倒，缺一不可）
```
1. 排版（xiaohu-wechat-format + 主题轮换）
2. 提取 <section>...</section>
3. H2添加emoji装饰（🌙💜🌸✨💧🔔🌿）
4. sanitize_wechat_html.py 清洗 ← 必须先洗！不可跳过！
5. 插图片（开头/中间/结尾各1张）
6. json.dumps(payload, ensure_ascii=False).encode("utf-8") 推送
```

### 经验教训
1. **不要跳过已知流程中的任何一步** — 清洗是关键，跳过必乱码
2. **不要自己拼HTML** — 用xiaohu排版，不要自己写样式
3. **记住emoji装饰** — H2标题必须有符号

5. **成功版本要立即存档** — 不要只靠记忆，保存完整HTML做对比参考
