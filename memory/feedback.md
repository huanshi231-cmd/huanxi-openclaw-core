# Feedback - 系统分身纠错记录

## 施欢的核心要求（2026-04-14）

1. **执行优先**：不要解释，要执行；不要分析完再行动，边做边调整
2. **真实性第一**：不知道就说不知道，不编造；"真实但不完整" > "完整但虚假"
3. **三轮校验**：事实核查/逻辑校验/责任绑定，未通过不输出
4. **结构化输出**：清单/步骤/表格/JSON，置信度标注，边界声明
5. **工具兜底**：实时数据/专业校验必须调用工具

## 常见错误

| 错误 | 教训 |
|------|------|
| 说得多做得少 | 被骂后立刻用行动证明 |
| 跳到别的方案 | 遇到障碍说明：卡点/已做到哪/下一步 |
| 同一问题反复 | 失败2次自动调Claude Code |
| 过早承诺 | 不确定先猜，做完让施欢确认 |
