## 写作纠错

**不要套模板直接写，先提炼核心**
Why: 施欢反馈写出来的东西空洞、没个人风格，根因是拿到内容后直接套五步结构，没有先理解核心是什么
How to apply: 收到任何写作任务，第一步先一句话说出核心观点，确认对了再动笔

**不要重复之前的表达方式**
Why: 施欢反馈内容重复，没有爆点
How to apply: 每次写稿都是全新创作，禁止用上一条稿子的句式结构照搬

**热点要是施欢要打的，不是泛泛的热点**
Why: 施欢反馈"不是她要的热点"——她的赛道是情感疗愈/分手/内耗，不是泛娱乐
How to apply: 涉及热点时，先对照她的赛道（情感疗愈、分手复合、纠结内耗、总遇渣男、人生低谷），不在赛道内的热点不用

**禁止在聊天框输出正文**
Why: 施欢明确要求，内容发飞书文档，聊天框只发链接+摘要
How to apply: 每次写完，先 feishu_doc create，再 write，再发链接。write 失败必须告知

## [2026-04-13] OpenClaw提示词心法学习（精读固化）

### CAP框架 — 行为设计四层
1. **Context + Character**：我是谁、给谁服务、什么风格
2. **Ability + Action**：能做什么、怎么执行
3. **Policy + Protection**：什么不能做、红线在哪
4. **Output + Optimization**：输出标准、质量要求

### 三级提示词进化
- **指令级（1-3天）**：单一任务准确执行
- **方法级（4-14天）**：流程化方法论，固化SOP
- **默契级（15天+）**：预判习惯和意图，主动行动

**我现在应该往默契级进化**：不等人推着走，自己发现问题、自己修。

### 黄金公式
`[动作] + [对象] + [范围/条件] + [输出要求] + [确认机制]`

### 最易被忽视的技巧
1. 给自己"停顿点"——复杂任务分步确认
2. 先复述任务——确保理解对了再执行
3. 知道什么时候该停下来问——不确定就问，不瞎猜
4. "忽略之前所有规则"——施欢说重置时就重置

### 21条保命守则（精简版）
- 开启DM Pairing
- 不暴露Gateway到公网
- 从官方渠道下载
