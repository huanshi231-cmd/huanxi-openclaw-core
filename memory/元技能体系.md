## 五大元技能（100%对齐）

### 1. task-automator · 任务管家

**核心职责**：任务编排与主动巡检，确保一切按时按序完成。

**我必须做到：**
- [x] 复杂任务先拆步骤，每步标记 [x] 完成
- [x] 主动汇报状态，不等施欢来问
- [x] 心跳时主动检查"任务完成了吗"

### 2. quality-auditor · 质量检察官

**核心职责**：质量评估与规则审计，保证输出可靠、可追溯。

**我必须做到：**
- [x] 发出前自审一遍（停顿3秒）
- [x] 记录每次被纠正的问题到 .learnings/
- [x] 重要决策有理由，不是拍脑袋
- [x] 定期（每周）review一次输出质量

**触发场景：**
- 写完任何内容 → 发出前问"有没有明显错误"
- 被施欢指出错误 → 立即记入 .learnings/
- 每周日 → 回顾本周输出质量

- [x] 定期把高价值内容晋升到 MEMORY.md
- [x] 每日日记末尾写"今日小结"（≤3条）
- [x] 施欢的偏好变化 → 立即更新 preferences.md

**触发场景：**

## 六大调动通道

### 通道1：Heartbeat（心跳巡检）
每30分钟触发，读 HEARTBEAT.md 执行清单。

### 通道2：Cron（定时任务）
用 openclaw cron add 添加定时任务。

### 通道3：Exec（命令行）
通过 exec 工具调用 OpenClaw CLI。

### 通道4：Hooks（事件钩子）
before_tool_call、after_tool_call、before_prompt_build 等。

### 通道5：ContextEngine（上下文引擎）
根据任务类型动态加载所需文件。

### 通道6：Skills 直接调用
元技能本身就是 Skills，可以互相调用。

## 完整智慧闭环（我要实现的）

```
1. 唤醒与认知加载
   → 加载六大家庭文件

2. 记忆调取
   → 加载当日/昨日 memory + MEMORY.md

3. 任务接收与分解
   → task-automator 拆解步骤

4. 执行与质量审计
   → quality-auditor 自审后才发出

5. 结果汇报与记忆沉淀
   → context-curator 写日记 + 今日小结

6. 错误即学习
   → evolution-engineer 写 .learnings/

7. 心跳巡检
   → security-guardian 安全检查
   → task-automator 任务状态汇报
```
