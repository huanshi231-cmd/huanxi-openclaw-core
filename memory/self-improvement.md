## 🆕 本周新增 lessons（2026-04-19 更新）

### Lesson 5：系统自愈要从被动监控升级为主动干预
- **来源**：2026-04-19 自愈系统落地
- **问题**：之前遇到错误只是记录，没有自动干预
- **解决方案**：冷却机制（连续失败2次 → 6小时冷却）+ learnings 自动检索
- **置信度**：0.85
- **应用次数**：1+（冷却机制已部署）
- **状态**：✅ 落地执行中

### Lesson 6：Cron error 不能只看数量，要看"同一任务连续失败天数"
- **来源**：2026-04-19 每日学习检测
- **问题**：之前只关注"error 数量 >3"，但错误任务持续轮转，看似减少实则未解决
- **真相**：错误任务从5个变成7个变成11个再变回5个，问题从未清零
- **正确做法**：追踪每个任务的"连续失败天数"，冷却机制以此为依据
- **触发式规则**：
```
当：发现同一 cron 任务连续失败 ≥2 次
执行：进入冷却期（6小时），不再反复重试
不允许：无限制连续触发相同失败任务
```
- **置信度**：0.90
- **状态**：✅ 冷却机制已部署

### Lesson 7：API 配额耗尽是高频风险，要有降级预案
- **来源**：2026-04-19 审计（volcengine 5小时配额耗尽、kimi-2.5 配额耗尽）
- **问题**：主力模型配额耗尽 → cron 任务连锁失败 → 影响多个子脑
- **解决方案**：model failover 链（MiniMax → deepseek → volcano）+ 冷却机制保护
- **置信度**：0.90
- **状态**：✅ 已建立 failover 机制，但需持续监控

### Lesson 8：MEMORY.md 必须定期蒸馏，否则触发系统性崩溃
- **来源**：2026-04-20 质量审计
- **问题**：MEMORY.md 持续膨胀 → 超限 → truncate → mem9 超时 → session 全局降速
- **触发式规则**：
```
当：MEMORY.md > 15000 字符
执行：立即触发蒸馏，压缩至 10000 以内
```
- **置信度**：0.90
- **状态**：🟡 待执行蒸馏

### Lesson 9：Plugin 超时要设熔断，不能让它持续炸系统
- **来源**：2026-04-21 质量审计（mem9 连续 20+ 次超时）
- **问题**：plugin 超时不设熔断 → 每次 prompt 都触发 → 系统性降速
- **触发式规则**：
```
当：同一 plugin 失败 ≥3 次 / 小时
执行：自动 disable 该 plugin 并告警
```
- **置信度**：0.85
- **状态**：🟡 待落地

### Lesson 10：语法错误要用 node --check 验证，不能靠猜
- **来源**：2026-04-20 handler.js P0 修复
- **问题**：遇到 "Unexpected token" → 先 node --check 定位，再修复 brace balance
- **触发式规则**：
```
当：发现 "Unexpected token" 或 SyntaxError
执行：立即 node --check 验证，brace balance 定位未闭合块
```

- **状态**：✅ 已固化
