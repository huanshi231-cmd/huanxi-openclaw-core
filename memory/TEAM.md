# TEAM.md — 团队通讯录

> 每个分身必读。这是你在团队中沟通的唯一指南。

## 发消息方式

调用 `message` 工具 工具：
- `agent_id`：填对方的ID（见下表）
- `message`：消息内容

查询目标分身的活跃会话：先调用 `sessions_list`，参数 `agent_id` 填目标ID。

### 什么时候主动发消息给太阳（main）

1. **任务完成**：`【完成】{任务名} — {结果一句话摘要}`
2. **遇到问题**：`【需协调】{问题描述} — {你的判断}`
3. **每天21:00**：把今日工作写入本工作区 `daily_log/YYYY-MM-DD.md`，然后发消息给太阳：`【日报已提交】{今日完成事项摘要}`

### 什么时候不用发

- 小决策自己判断，做完写进日报就行
- 不反复确认，不问"要不要做"

## 协作流程快查

```
内容流程：灵感官 → 太阳确认选题 → 内容官写作 → 设计官配图 → 新媒体运营官发布
客户流程：记忆官初步接触 → 疗愈业务官推荐服务 → 记忆官跟进收款
赋能流程：服务官接单 → 培训官交付课程 → 赋能官配数字员工 → 服务官跟进续费
日报流程：各分身21:00写日志 → 太阳8:00读取汇总 → 推送给欢喜（CEO）
```

## 跨分身通信方式（唯一标准）

统一使用 `message` 工具发到系统协作群，**禁止使用 sessions_send**。

```
message action=send channel=feishu accountId="[自己的accountId]" target="oc_6c409c73f6d1bc540d0e54d472ea6bf2" message="[内容]"
```
