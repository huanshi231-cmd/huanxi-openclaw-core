# web-access 落地计划

## 这个 skill 是什么
给 AI 装上完整联网能力，三层通道调度：
- WebSearch / WebFetch / curl / Jina — 轻量抓取
- CDP 浏览器 — 直连你的 Chrome，携带登录态，支持动态页面、视频截帧
- 并行分治 — 多目标分发给子 Agent 并行跑

## 核心价值（对我们系统的意义）

| 场景 | 现状 | 有了之后 |
|------|------|---------|
| 微博/知乎热榜 | 依赖第三方 API | 直接登录抓，最实时 |
| 微信公众号文章 | 需要草稿箱 API | 直接读读者文章页 |
| 小红书内嵌图片 | 一直无法内嵌飞书 | 通过 Chrome 直取图片 URL |
| 视频内容分析 | 只能分析文字 | 截帧分析视频画面 |
| 站点操作经验 | 每次重头学 | 按域名积累，跨 session 复用 |

## 前置条件检查

| 条件 | 状态 | 操作 |
|------|------|------|
| Node.js ≥ 22 | ✅ v25.8.0 已满足 | — |
| Chrome remote debugging | ❌ 未开启 | 需配置 Chrome 启动参数 |
| 备用账号 | ⚠️ 建议 | 小红书等平台建议用小号 |

## 安装步骤

**Step 1** — 克隆到本地 skill 目录
**Step 2** — 运行依赖检查脚本
**Step 3** — 配置 Chrome remote debugging（脚本自动指导）
**Step 4** — 验证连通性
**Step 5** — 定义第一批目标站点

## 第一批可落地任务（优先级排序）

### P0（立即可做）
1. **热榜实时采集**：微博/知乎热榜直接抓，替换现有 API 调用
2. **微信公众号文章**：直接读读者文章页，不需要草稿箱

### P1（需要配置 Chrome）
3. **小红书图片内嵌**：通过 CDP 直取图片 URL，尝试内嵌飞书
4. **视频内容分析**：对抖音/小红书视频截帧，分析画面内容

### P2（后续扩展）
5. **站点经验积累**：学习各平台的反爬机制，自动记录最佳操作
6. **批量内容采集**：并行分治，同时抓多个目标

## 风险提示
- CDP 直连 Chrome = 操作在用户已登录的浏览器里，建议小红书等平台用备用账号
- Chrome 需保持运行 remote debugging port，脚本会指导配置
